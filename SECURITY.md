# Security Policy

## Supported Versions

Security fixes target the latest `main` branch.

## Reporting a Vulnerability

Do not open a public issue for suspected vulnerabilities. Use GitHub Security Advisories when available, or contact the maintainer privately.

Do not include real server serial numbers, customer data, API keys, signing files, certificates, or production endpoints in reports unless the maintainer explicitly requests a secure transfer method.

This project does not publish a fixed response SLA in its first public release.
